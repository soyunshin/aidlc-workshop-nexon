## 유닛별 스토리 커버리지

### Unit 1: database-core
- **관련 스토리**: 전체 (모든 스토리의 데이터 모델 기반)
- **역할**: 데이터 모델 정의, DB 인프라 제공

### Unit 2: auth-backend
- **관련 스토리**: US-01 (테이블 로그인), US-06 (관리자 로그인)
- **역할**: 인증 API 제공

### Unit 3: menu-backend
- **관련 스토리**: US-02 (메뉴 조회), US-09 (메뉴 관리)
- **역할**: 메뉴 CRUD API 제공

### Unit 4: order-table-backend
- **관련 스토리**: US-04 (주문 생성), US-05 (주문 조회), US-07 (실시간 모니터링), US-08 (테이블 관리)
- **역할**: 주문/테이블/SSE API 제공

### Unit 5: customer-frontend
- **관련 스토리**: US-01 (자동 로그인 UI), US-02 (메뉴 UI), US-03 (장바구니), US-04 (주문 UI), US-05 (주문 내역 UI)
- **역할**: 고객 사용자 인터페이스

### Unit 6: admin-frontend
- **관련 스토리**: US-06 (로그인 UI), US-07 (대시보드 UI), US-08 (테이블 관리 UI), US-09 (메뉴 관리 UI)
- **역할**: 관리자 사용자 인터페이스

### Unit 7: integration-testing
- **관련 스토리**: 전체 (모든 스토리의 E2E 검증)
- **역할**: 통합 테스트, 전체 플로우 검증
