# 사용자 스토리 - 테이블오더 서비스

## 스토리 구성 방식
- **분류**: User Journey-Based (사용자 워크플로우 순서)
- **크기**: Epic 수준 (주요 기능 단위)
- **수용 기준**: 간결 (Given/When/Then, 핵심 시나리오 2~3개)

---

## 고객 Journey

### US-01: 테이블 자동 로그인

**As a** 고객 (김고객)  
**I want to** 테이블에 앉으면 별도 로그인 없이 바로 주문 화면을 볼 수 있다  
**So that** 대기 시간 없이 즉시 메뉴를 탐색하고 주문할 수 있다

**Acceptance Criteria:**
- Given 태블릿에 로그인 정보가 저장되어 있을 때, When 브라우저를 열면, Then 자동으로 로그인되어 메뉴 화면이 표시된다
- Given 저장된 로그인 정보가 없을 때, When 브라우저를 열면, Then 초기 설정 화면이 표시된다
- Given 세션이 만료되었을 때, When 페이지에 접근하면, Then 저장된 정보로 자동 재인증이 수행된다

**Persona**: 김고객  
**Priority**: High  

---

### US-02: 메뉴 조회 및 탐색

**As a** 고객 (김고객)  
**I want to** 카테고리별로 메뉴를 탐색하고 상세 정보를 확인할 수 있다  
**So that** 원하는 음식을 쉽게 찾아 주문할 수 있다

**Acceptance Criteria:**
- Given 메뉴 화면에 있을 때, When 카테고리를 선택하면, Then 해당 카테고리의 메뉴 목록이 카드 형태로 표시된다
- Given 메뉴 목록이 표시될 때, When 메뉴 카드를 확인하면, Then 메뉴명, 가격, 설명, 이미지가 표시된다
- Given 다른 카테고리를 보고 싶을 때, When 카테고리 탭을 탭하면, Then 해당 카테고리로 빠르게 이동한다

**Persona**: 김고객  
**Priority**: High  

---

### US-03: 장바구니 관리

**As a** 고객 (김고객)  
**I want to** 원하는 메뉴를 장바구니에 담고 수량을 조절할 수 있다  
**So that** 주문 전에 선택한 메뉴를 확인하고 수정할 수 있다

**Acceptance Criteria:**
- Given 메뉴 화면에서 메뉴를 선택할 때, When 장바구니에 추가하면, Then 장바구니에 메뉴가 추가되고 총 금액이 업데이트된다
- Given 장바구니에 메뉴가 있을 때, When 수량을 변경하면, Then 수량과 총 금액이 실시간으로 재계산된다
- Given 장바구니에 메뉴가 있을 때, When 페이지를 새로고침하면, Then 장바구니 내용이 유지된다

**Persona**: 김고객  
**Priority**: High  

---

### US-04: 주문 생성

**As a** 고객 (김고객)  
**I want to** 장바구니의 메뉴를 확인하고 주문을 확정할 수 있다  
**So that** 선택한 음식이 주방에 전달되어 준비될 수 있다

**Acceptance Criteria:**
- Given 장바구니에 메뉴가 있을 때, When 주문 확정 버튼을 누르면, Then 주문이 생성되고 주문 번호가 표시된 후 5초 뒤 메뉴 화면으로 돌아간다
- Given 주문이 성공했을 때, When 메뉴 화면으로 돌아오면, Then 장바구니가 비워져 있다
- Given 주문 처리 중 오류가 발생했을 때, When 에러가 반환되면, Then 에러 메시지가 표시되고 장바구니는 유지된다

**Persona**: 김고객  
**Priority**: High  

---

### US-05: 주문 내역 조회

**As a** 고객 (김고객)  
**I want to** 현재 테이블에서 주문한 내역과 상태를 확인할 수 있다  
**So that** 주문이 정상적으로 접수되었는지, 현재 준비 상태가 어떤지 알 수 있다

**Acceptance Criteria:**
- Given 주문 내역 화면에 접근할 때, When 주문 목록이 표시되면, Then 현재 세션의 주문만 시간순으로 표시된다
- Given 주문 목록이 표시될 때, When 각 주문을 확인하면, Then 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)가 표시된다

**Persona**: 김고객  
**Priority**: Medium  

---

## 관리자 Journey

### US-06: 매장 인증

**As a** 매장 관리자 (박사장)  
**I want to** 매장 식별자와 계정 정보로 로그인할 수 있다  
**So that** 매장 관리 시스템에 안전하게 접근할 수 있다

**Acceptance Criteria:**
- Given 로그인 화면에서, When 올바른 매장 식별자/사용자명/비밀번호를 입력하면, Then 관리자 대시보드로 이동하고 16시간 세션이 시작된다
- Given 로그인된 상태에서, When 브라우저를 새로고침하면, Then 세션이 유지되어 재로그인 없이 사용 가능하다
- Given 잘못된 인증 정보를 입력할 때, When 반복 시도하면, Then 로그인 시도가 제한된다

**Persona**: 박사장  
**Priority**: High  

---

### US-07: 실시간 주문 모니터링

**As a** 매장 관리자 (박사장)  
**I want to** 들어오는 주문을 실시간으로 확인하고 상태를 변경할 수 있다  
**So that** 주문을 놓치지 않고 효율적으로 처리할 수 있다

**Acceptance Criteria:**
- Given 대시보드에 있을 때, When 새 주문이 들어오면, Then 2초 이내에 해당 테이블 카드에 주문이 표시되고 시각적으로 강조된다
- Given 주문 카드가 표시될 때, When 상태 변경 버튼을 누르면, Then 주문 상태가 대기중→준비중→완료로 변경된다
- Given 테이블 카드를 클릭할 때, When 상세 보기가 열리면, Then 해당 테이블의 전체 주문 메뉴 목록이 표시된다

**Persona**: 박사장  
**Priority**: High  

---

### US-08: 테이블 관리

**As a** 매장 관리자 (박사장)  
**I want to** 테이블 설정, 주문 삭제, 이용 완료 처리를 할 수 있다  
**So that** 테이블 운영을 효율적으로 관리하고 다음 고객을 맞을 준비를 할 수 있다

**Acceptance Criteria:**
- Given 테이블에 주문이 있을 때, When 이용 완료 버튼을 누르면, Then 확인 팝업 후 해당 세션의 주문이 과거 이력으로 이동하고 테이블이 리셋된다
- Given 잘못된 주문이 있을 때, When 주문 삭제 버튼을 누르면, Then 확인 팝업 후 주문이 삭제되고 총 주문액이 재계산된다
- Given 과거 내역을 확인하고 싶을 때, When 과거 내역 버튼을 누르면, Then 해당 테이블의 과거 주문 목록이 시간 역순으로 표시된다

**Persona**: 박사장  
**Priority**: High  

---

### US-09: 메뉴 관리

**As a** 매장 관리자 (박사장)  
**I want to** 메뉴를 등록, 수정, 삭제하고 노출 순서를 조정할 수 있다  
**So that** 매장의 메뉴를 최신 상태로 유지하고 고객에게 정확한 정보를 제공할 수 있다

**Acceptance Criteria:**
- Given 메뉴 관리 화면에서, When 새 메뉴를 등록하면, Then 메뉴명, 가격, 설명, 카테고리, 이미지 URL이 저장되고 고객 화면에 반영된다
- Given 기존 메뉴가 있을 때, When 메뉴 정보를 수정하면, Then 변경 사항이 즉시 저장되고 고객 화면에 반영된다
- Given 메뉴 목록이 있을 때, When 노출 순서를 변경하면, Then 고객 화면에서 변경된 순서로 메뉴가 표시된다

**Persona**: 박사장  
**Priority**: Medium
