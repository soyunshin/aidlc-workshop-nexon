# 사용자 페르소나 - 테이블오더 서비스

## Persona 1: 고객 (Customer)

| 항목 | 내용 |
|------|------|
| **이름** | 김고객 |
| **역할** | 매장 방문 고객 |
| **설명** | 매장 테이블에 앉아 태블릿을 통해 음식을 주문하는 일반 고객 |
| **목표** | 빠르고 편리하게 메뉴를 확인하고 주문하기 |
| **동기** | 직원 호출 없이 즉시 주문, 주문 상태 확인 |
| **기술 수준** | 다양 (스마트폰/태블릿 기본 사용 가능) |
| **사용 환경** | 매장 내 테이블 태블릿 (브라우저 기반) |
| **핵심 니즈** | 직관적 UI, 빠른 주문 프로세스, 주문 현황 확인 |
| **불만 요소** | 복잡한 인터페이스, 느린 응답, 주문 누락 |

### 주요 시나리오
1. 테이블에 앉아 태블릿으로 메뉴를 탐색
2. 원하는 메뉴를 장바구니에 담고 주문
3. 추가 주문이 필요할 때 다시 메뉴에서 선택
4. 주문 상태를 확인하며 대기

## Persona 2: 매장 관리자 (Admin)

| 항목 | 내용 |
|------|------|
| **이름** | 박사장 |
| **역할** | 매장 사장님 (1인 운영) |
| **설명** | 매장의 모든 운영을 관리하는 사장님. 주문 접수, 메뉴 관리, 테이블 관리를 담당 |
| **목표** | 효율적인 매장 운영, 주문 누락 방지, 매출 관리 |
| **동기** | 인건비 절감, 주문 실수 감소, 운영 효율화 |
| **기술 수준** | 중간 (기본적인 웹 서비스 사용 가능) |
| **사용 환경** | 카운터/주방 근처 PC 또는 태블릿 (브라우저 기반) |
| **핵심 니즈** | 실시간 주문 확인, 간편한 메뉴 관리, 테이블 상태 파악 |
| **불만 요소** | 주문 놓침, 복잡한 관리 도구, 시스템 장애 |

### 주요 시나리오
1. 영업 시작 시 시스템 로그인
2. 실시간으로 들어오는 주문 확인 및 상태 변경
3. 메뉴 추가/수정/삭제
4. 고객 퇴장 시 테이블 이용 완료 처리
5. 과거 주문 내역 확인
