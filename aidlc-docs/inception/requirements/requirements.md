## Intent Analysis

| 항목 | 내용 |

| **사용자 요청** | 테이블오더 서비스 구축 (디지털 주문 시스템) |

## 기능 요구사항 (Functional Requirements)

### FR-01: 고객용 - 테이블 태블릿 자동 로그인 및 세션 관리

| ID | 요구사항 |
|----|----------|
| FR-01-01 | 관리자가 초기 설정 시 매장 식별자, 테이블 번호, 테이블 비밀번호를 입력 |
| FR-01-02 | 로그인 정보를 로컬 스토리지에 저장 |
| FR-01-03 | 저장된 정보로 자동 로그인 수행 |
| FR-01-04 | 세션 만료 시 저장된 정보로 자동 재인증 |

### FR-02: 고객용 - 메뉴 조회 및 탐색

| ID | 요구사항 |
|----|----------|
| FR-02-01 | 메뉴 화면이 기본 화면으로 표시 |
| FR-02-02 | 카테고리별 메뉴 분류 및 표시 |
| FR-02-03 | 메뉴 상세 정보 표시 (메뉴명, 가격, 설명, 이미지) |
| FR-02-04 | 카테고리 간 빠른 이동 지원 |
| FR-02-05 | 카드 형태 레이아웃, 터치 친화적 버튼 (최소 44x44px) |

### FR-03: 고객용 - 장바구니 관리

| ID | 요구사항 |
|----|----------|
| FR-03-01 | 메뉴 추가/삭제 기능 |
| FR-03-02 | 수량 조절 (증가/감소) |
| FR-03-03 | 총 금액 실시간 계산 |
| FR-03-04 | 장바구니 비우기 |
| FR-03-05 | 로컬 저장 (페이지 새로고침 시에도 유지) |
| FR-03-06 | 서버 전송은 주문 확정 시에만 수행 |

### FR-04: 고객용 - 주문 생성

| ID | 요구사항 |
|----|----------|
| FR-04-01 | 주문 내역 최종 확인 화면 |
| FR-04-02 | 주문 확정 버튼 |
| FR-04-03 | 주문 성공 시: 주문 번호 표시, 장바구니 비우기, 5초 후 메뉴 화면 리다이렉트 |
| FR-04-04 | 주문 실패 시: 에러 메시지 표시, 장바구니 유지 |
| FR-04-05 | 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID |

### FR-05: 고객용 - 주문 내역 조회

| ID | 요구사항 |
|----|----------|
| FR-05-01 | 주문 시간 순 정렬 |
| FR-05-02 | 주문별 상세 정보 표시 (주문 번호, 시각, 메뉴/수량, 금액, 상태) |
| FR-05-03 | 주문 상태: 대기중/준비중/완료 |

| ID | 요구사항 |
|----|----------|
| NFR-01-01 | SSE를 통한 실시간 주문 알림 2초 이내 전달 |

| NFR-01-03 | 동시 접속 10개 테이블 + 관리자 1명 지원 |

### NFR-04: 사용성

| ID | 요구사항 |
|----|----------|
| NFR-04-01 | 터치 친화적 UI (최소 44x44px 터치 타겟) |
| NFR-04-02 | 직관적인 카드 기반 레이아웃 |
| NFR-04-03 | 명확한 시각적 계층 구조 |

## 제외 범위 (Out of Scope)

다음 기능은 이번 MVP에서 구현하지 않습니다:
- 결제 처리 (카드, 현금, PG사 연동)
- 복잡한 인증 (OAuth, SNS 로그인, 2FA)
- 이미지 업로드/리사이징
- 알림 시스템 (푸시, SMS, 이메일)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (데이터 분석, 매출 리포트, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어)

## 데이터 모델 개요

### 핵심 엔티티
- **Store**: 매장 정보 (단일 매장)
- **Admin**: 관리자 계정 (시드 데이터로 생성)
- **Table**: 테이블 정보 (번호, 비밀번호, 현재 세션 ID)
- **TableSession**: 테이블 세션 (시작 시각, 종료 시각, 상태)
- **Category**: 메뉴 카테고리
- **MenuItem**: 메뉴 항목 (이름, 가격, 설명, 이미지 URL, 노출 순서)
- **Order**: 주문 (테이블, 세션, 상태, 총 금액, 주문 시각)
- **OrderItem**: 주문 항목 (메뉴, 수량, 단가)
- **OrderHistory**: 과거 주문 이력 (세션 종료 시 이동)

## 용어 정의

| 용어 | 정의 |
|------|------|
| 테이블 세션 | 고객이 첫 주문 시작부터 이용 완료 처리까지의 기간 |
| 이용 완료 | 관리자가 테이블 세션을 종료하는 행위 |
| SSE | Server-Sent Events, 서버에서 클라이언트로 단방향 실시간 데이터 전송 |
| JWT | JSON Web Token, 인증 토큰 |
