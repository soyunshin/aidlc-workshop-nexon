# Business Logic Model - Unit 5: Customer Frontend

## 개요

고객 프론트엔드의 비즈니스 로직은 클라이언트 측에서 처리되는 로직과 서버 API 호출로 구분됩니다.

---

## 1. 자동 로그인 로직

```
시작
  │
  ├── localStorage에 저장된 인증 정보 확인
  │     ├── 있음 → JWT 토큰 유효성 확인
  │     │           ├── 유효 → 메뉴 화면 표시
  │     │           └── 만료 → 저장된 credentials로 재인증 API 호출
  │     │                       ├── 성공 → 새 토큰 저장 → 메뉴 화면
  │     │                       └── 실패 → 초기 설정 화면 표시
  │     └── 없음 → 초기 설정 화면 표시
  │
종료
```

**클라이언트 측 로직:**
- JWT 토큰 만료 시간 체크 (디코딩하여 exp 확인)
- localStorage에 store_id, table_number, password 저장
- 자동 재인증 시 저장된 정보로 POST /api/table/login 호출

---

## 2. 메뉴 조회 로직

```
메뉴 화면 진입
  │
  ├── GET /api/menu/categories → 카테고리 목록 로드
  ├── GET /api/menu/items → 전체 메뉴 로드
  │
  ├── 카테고리 탭 클릭
  │     └── 해당 카테고리 메뉴만 필터링 (클라이언트 측)
  │
  └── 메뉴 카드 클릭
        └── 장바구니에 추가 (수량 1)
```

**클라이언트 측 로직:**
- 카테고리별 필터링은 클라이언트에서 수행 (전체 데이터 한 번에 로드)
- 메뉴 데이터 캐싱 (페이지 이동 시 재요청 방지)

---

## 3. 장바구니 로직

```
장바구니 조작
  │
  ├── 메뉴 추가
  │     ├── 이미 있는 메뉴 → 수량 +1
  │     └── 새 메뉴 → 수량 1로 추가
  │
  ├── 수량 변경
  │     ├── 증가 → 수량 +1
  │     └── 감소 → 수량 -1 (0이면 삭제)
  │
  ├── 메뉴 삭제 → 장바구니에서 제거
  │
  ├── 장바구니 비우기 → 전체 삭제
  │
  └── 총 금액 계산
        └── SUM(각 항목의 단가 × 수량)
```

**클라이언트 측 로직:**
- 모든 장바구니 조작은 클라이언트 상태(Context)에서 처리
- localStorage에 동기화 (새로고침 시 유지)
- 서버 전송은 주문 확정 시에만 수행

---

## 4. 주문 생성 로직

```
주문 확정
  │
  ├── 장바구니 비어있음 체크 → 비어있으면 에러
  │
  ├── 주문 데이터 구성
  │     ├── store_id (저장된 인증 정보에서)
  │     ├── table_id (저장된 인증 정보에서)
  │     ├── session_id (JWT 토큰에서)
  │     └── items: [{menu_item_id, quantity, unit_price}]
  │
  ├── POST /api/orders 호출
  │     ├── 성공 (201)
  │     │     ├── 장바구니 비우기 (Context + localStorage)
  │     │     ├── 주문 번호 표시 (성공 페이지)
  │     │     └── 5초 카운트다운 → 메뉴 화면 리다이렉트
  │     └── 실패 (4xx/5xx)
  │           ├── 에러 메시지 표시
  │           └── 장바구니 유지
  │
종료
```

---

## 5. 주문 내역 조회 로직

```
주문 내역 화면
  │
  ├── GET /api/orders (JWT 토큰으로 세션 식별)
  │     └── 현재 세션의 주문 목록 반환
  │
  ├── 주문 목록 표시 (시간순 정렬)
  │     └── 각 주문: 번호, 시각, 메뉴/수량, 금액, 상태
  │
  └── 상태 표시
        ├── 대기중 (pending) → 노란색
        ├── 준비중 (preparing) → 파란색
        └── 완료 (completed) → 초록색
```
