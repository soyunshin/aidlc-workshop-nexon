# NFR Design Patterns - Unit 5: Customer Frontend

## 1. 성능 패턴

### 1.1 데이터 캐싱 패턴
| 항목 | 설계 |
|------|------|
| **패턴** | Client-Side Cache (메모리 + localStorage) |
| **적용 대상** | 메뉴 데이터, 카테고리 데이터 |
| **전략** | 앱 로드 시 전체 메뉴 1회 로드 → 메모리 캐시 → 카테고리 필터링은 클라이언트 측 |
| **무효화** | 페이지 새로고침 시 재로드 (실시간 메뉴 변경은 MVP에서 미지원) |

### 1.2 지연 로딩 패턴
| 항목 | 설계 |
|------|------|
| **패턴** | Route-based Code Splitting |
| **적용 대상** | 관리자 페이지 (고객이 접근하지 않는 코드) |
| **구현** | React.lazy + Suspense로 admin 라우트 동적 import |
| **효과** | 고객 초기 로드 번들 사이즈 감소 |

### 1.3 낙관적 업데이트 패턴
| 항목 | 설계 |
|------|------|
| **패턴** | Optimistic UI Update |
| **적용 대상** | 장바구니 조작 (추가, 수량 변경, 삭제) |
| **구현** | 서버 호출 없이 즉시 UI 반영 (장바구니는 클라이언트 전용) |
| **효과** | 0ms 응답 체감 |

## 2. 신뢰성 패턴

### 2.1 에러 바운더리 패턴
| 항목 | 설계 |
|------|------|
| **패턴** | Error Boundary (React) |
| **적용 대상** | 앱 전체 (최상위) + 페이지 단위 |
| **동작** | 컴포넌트 렌더링 에러 시 fallback UI 표시, 앱 크래시 방지 |
| **로깅** | 에러 정보를 콘솔에 기록 (프로덕션에서는 에러 리포팅 서비스로 전송) |

### 2.2 자동 재시도 패턴
| 항목 | 설계 |
|------|------|
| **패턴** | Retry with Exponential Backoff |
| **적용 대상** | API 호출 실패 (네트워크 에러, 5xx) |
| **구현** | Axios 인터셉터에서 최대 2회 재시도 (1초, 3초 간격) |
| **제외** | 4xx 에러 (클라이언트 에러)는 재시도하지 않음 |

### 2.3 자동 재인증 패턴
| 항목 | 설계 |
|------|------|
| **패턴** | Token Refresh on 401 |
| **적용 대상** | JWT 만료 시 |
| **동작** | 401 응답 → 저장된 credentials로 재로그인 → 원래 요청 재시도 |
| **실패 시** | 설정 화면으로 리다이렉트 |

## 3. 보안 패턴

### 3.1 토큰 관리 패턴
| 항목 | 설계 |
|------|------|
| **패턴** | Secure Token Storage (SPA) |
| **저장소** | localStorage (SPA에서 httpOnly 쿠키 불가) |
| **보호** | XSS 방지로 토큰 탈취 위험 최소화 |
| **만료** | JWT exp 클레임 체크, 만료 시 자동 재인증 |

### 3.2 입력 새니타이징 패턴
| 항목 | 설계 |
|------|------|
| **패턴** | Input Sanitization |
| **적용 대상** | 사용자 입력 필드 (설정 화면) |
| **구현** | React의 기본 XSS 방지 (JSX 자동 이스케이프), dangerouslySetInnerHTML 사용 금지 |

### 3.3 HTTPS 강제 패턴
| 항목 | 설계 |
|------|------|
| **패턴** | Transport Security |
| **구현** | 프로덕션 배포 시 HTTPS 전용, HSTS 헤더 (서버/CDN 설정) |
| **개발** | localhost에서는 HTTP 허용 (Vite dev server) |

## 4. 사용성 패턴

### 4.1 로딩 상태 패턴
| 항목 | 설계 |
|------|------|
| **패턴** | Loading State Feedback |
| **구현** | 스피너 (데이터 로딩), 버튼 비활성화 (폼 제출 중) |
| **적용** | 모든 API 호출 컴포넌트 |

### 4.2 오프라인 내구성 패턴
| 항목 | 설계 |
|------|------|
| **패턴** | Local Persistence |
| **적용 대상** | 장바구니 데이터 |
| **구현** | useEffect로 상태 변경 시 localStorage 동기화 |
| **효과** | 네트워크 끊김/새로고침 시에도 장바구니 유지 |
